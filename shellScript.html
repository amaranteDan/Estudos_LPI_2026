Ciclo de vida do comando: O Shell processa a linha em etapas antes de executar qualquer coisa. Entender essa ordem evita 80% dos bugs.
#EtapaExemplo
1Leitura da linhaecho ${A} *.txt
2Tokenizaçãodivide em palavras
3Expansão de chaves{a,b} → a b
4Expansão de til~ → /home/user
5Substituição de variáveis$VAR → valor
6Substituição de comandos$(cmd) → saída
7Expansão aritmética$((2+3)) → 5
8Glob (nomes de arquivo)*.txt → arquivos
9Execuçãochama o programa
bash
# Exemplo da ordem de expansão
DIR="*.txt"
ls $DIR
# 1. $DIR é expandido → *.txt
# 2. *.txt é expandido → lista de arquivos
# 3. ls recebe os nomes dos arquivos

echo "Hoje é $(date +%A)"
# $(date +%A) é executado ANTES do echo

Coringas (Wildcards / Glob) são metacaracteres expandidos pelo Shell antes de passar os argumentos ao comando. Não são regex!
CoringaSignificadoExemplo
*Qualquer sequência de caracteres (inclusive vazia)*.txt → todos os .txt
?Exatamente um caractere qualquerarq?.txt → arq1.txt, arqA.txt
[abc]Um caractere do conjunto[aeiou]* → começa com vogal
[a-z]Um caractere no intervalo[0-9]* → começa com dígito
[!abc]Um caractere fora do conjunto[!0-9]* → não começa com dígito
**Recursivo (com globstar ativo)**/*.sh → todos .sh em subpastas
bash — Exemplos práticos
ls *.log                  # todos os arquivos .log
ls relatorio_????.txt     # relatorio_ + 4 chars + .txt
ls [A-Z]*.sh              # scripts que começam com maiúscula
rm tmp[0-9][0-9]          # remove tmp00 até tmp99
mv [!.]* /backup/         # move tudo exceto arquivos ocultos

# Ativar globstar para recursão
shopt -s globstar
ls **/*.py                 # todos .py em qualquer subpasta

# Cuidado: se nenhum arquivo casar, o glob é passado literal
ls *.xyz                  # se não existir → erro "No such file"
# Para evitar: shopt -s nullglob
shopt -s nullglob
ls *.xyz                  # retorna vazio sem erro
Glob ≠ Regex: * no glob significa "qualquer coisa", mas em regex significa "zero ou mais do anterior". Não confunda ao usar com grep, sed ou awk.

bash — Sintaxe e exemplos

echo {a..e}              # → a b c d e

# Backup rápido (sufixo vazio + .bak)

bash — Produto Cartesiano (exercício)
# Combinação de múltiplas chaves = produto cartesiano

# → A1 A2 A3 B1 B2 B3

echo {x,y,z}{+,-}
# → x+ x- y+ y- z+ z-

# Criar combinações de arquivos de teste

# → test_unit_pass.log test_unit_fail.log
#    test_integ_pass.log test_integ_fail.log

# Aninhamento

# → a b1 b2 c
💡 Performance: Brace expansion é feita pelo próprio Shell, sem fork de processo. É muito mais rápido que usar um loop for para gerar sequências longas.
time mede o tempo de execução de um comando. Útil para comparar performance de scripts e identificar gargalos.
CampoSignificado
realTempo total decorrido (relógio de parede) — o que você sente esperando
userTempo de CPU gasto no espaço do usuário (seu processo)
sysTempo de CPU gasto no kernel (chamadas de sistema, I/O)
bash — Uso básico
time ls -lR /usr
# real    0m1.234s
# user    0m0.456s
# sys     0m0.778s

time sleep 2
# real    0m2.001s   ← quase tudo em "real" (processo dormindo)
# user    0m0.000s
# sys     0m0.001s

# Medir um script inteiro
time bash meu_script.sh

# Medir pipeline
time { find / -name "*.log" | wc -l; }
bash — time builtin vs \time externo
# time sem barra → builtin do bash (saída simples)
time grep -r "erro" /var/log

# \time → comando externo /usr/bin/time (mais detalhes)
\time -v grep -r "erro" /var/log
# Mostra: max RSS, page faults, context switches, etc.

# Redirecionar saída do time para arquivo
{ time meu_script.sh; } 2> tempo.txt

# Comparar dois métodos
echo "=== Método 1 ==="
time { sort big.txt | uniq -c; }

echo "=== Método 2 ==="
time { awk '{count[$0]++} END{for(k in count) print count[k], k}' big.txt; }
Interpretando os resultados:
• real >> user+sys → processo ficou esperando I/O ou sleep
• user >> sys → processamento intenso de CPU (cálculos, loops)
• sys >> user → muitas chamadas ao kernel (leitura de arquivos, rede)
🔢 bc — Calculadora de Precisão
▶

bc (Basic Calculator) suporta ponto flutuante, funções matemáticas e scripts. O Shell nativo só faz aritmética inteira.

bash — bc
# Ponto flutuante com scale
echo "scale=2; 10/3" | bc         # → 3.33
echo "scale=4; 22/7" | bc         # → 3.1428 (aprox. pi)

# Funções matemáticas (requer -l)
echo "sqrt(144)" | bc -l          # → 12.00000000000000000000
echo "s(3.14159)" | bc -l         # seno (radianos)
echo "l(10)" | bc -l              # logaritmo natural

# Potência e módulo
echo "2^10" | bc                  # → 1024
echo "17 % 5" | bc               # → 2

# Here String (mais limpo)
bc -l <<< "scale=6; 4*a(1)"      # → 3.141592 (pi via arctan)

# Capturar resultado em variável
RESULTADO=$(echo "scale=2; 100/3" | bc)
echo "Resultado: $RESULTADO"
🔀 tee — Duplicar Saída
▶

tee lê da stdin e escreve simultaneamente na stdout e em um arquivo. Ideal para logar e ver ao mesmo tempo.

bash — tee
# Salvar saída E ver no terminal
ls -la | tee listagem.txt

# Append (não sobrescreve)
echo "nova linha" | tee -a log.txt

# Gravar em múltiplos arquivos
echo "dados" | tee arq1.txt arq2.txt arq3.txt

# Usar no meio de um pipeline
cat dados.csv | tee backup.csv | grep "erro" | wc -l

# Salvar resultado de bc e mostrar
echo "scale=2; 355/113" | bc | tee resultado.txt
# → mostra 3.14 no terminal E salva em resultado.txt

# Com sudo (escrever em arquivo protegido)
echo "nova config" | sudo tee /etc/config.conf
📦 tar — Arquivamento e Extração
▶

tar (Tape ARchive) agrupa arquivos em um único arquivo. Combinado com gzip, bzip2 ou xz para compressão.

bash — tar: criar e extrair
# ── CRIAR ──────────────────────────────────────
tar -czf backup.tar.gz  pasta/    # gzip  (.tar.gz)
tar -cjf backup.tar.bz2 pasta/    # bzip2 (.tar.bz2)
tar -cJf backup.tar.xz  pasta/    # xz    (.tar.xz, melhor compressão)
tar -cf  backup.tar     pasta/    # sem compressão

# ── EXTRAIR ────────────────────────────────────
tar -xzf backup.tar.gz            # extrai no diretório atual
tar -xzf backup.tar.gz -C /tmp/   # extrai em /tmp/
tar -xf  backup.tar.xz            # auto-detecta compressão

# ── LISTAR ─────────────────────────────────────
tar -tzf backup.tar.gz            # lista conteúdo sem extrair
tar -tvf backup.tar.gz            # lista com detalhes (verbose)

# ── EXTRAIR ARQUIVO ESPECÍFICO ─────────────────
tar -xzf backup.tar.gz pasta/config.txt

# ── EXCLUIR PADRÃO AO CRIAR ────────────────────
tar -czf backup.tar.gz pasta/ --exclude="*.log" --exclude=".git"
Macete para lembrar as flags:
c = Create | x = eXtract | t = lisT
z = gZip | j = bzip2 | J = xz
f = File (sempre o último, seguido do nome)
v = Verbose | C = diretório de destino

sed (Stream Editor) processa texto linha a linha, aplicando comandos sem abrir um editor interativo.
bash — Comandos Básicos
sed -n '2,4p' arquivo.txt       # imprime linhas 2 a 4
sed '3d' arquivo.txt              # deleta linha 3
sed '2i\Linha inserida' arq.txt   # insere ANTES da linha 2
sed '2a\Linha após' arq.txt       # insere APÓS a linha 2
sed '2c\Linha nova' arq.txt       # troca a linha 2
sed -n '/erro/p' log.txt          # imprime linhas com "erro"
sed '/^$/d' arquivo.txt           # remove linhas em branco
bash — Substituição e Retrovisores
sed 's/foo/bar/g' arquivo.txt    # substitui todas as ocorrências
sed 's/foo/bar/2' arquivo.txt    # substitui só a 2ª ocorrência
sed 's/foo/bar/gi' arquivo.txt   # case-insensitive

# Delimitador alternativo (útil com caminhos)
sed 's|/usr/local|/opt|g' arq.txt

# Retrovisores: captura com \(...\) e referencia com \1
echo "João Silva" | sed 's/\(.*\) \(.*\)/\2, \1/'
# → Silva, João
Pattern Space: buffer de trabalho — contém a linha atual.
Hold Space: buffer auxiliar — armazena dados entre linhas.
bash
# Inverter ordem das linhas (tac manual)
sed -n '1!G; h; $p' arquivo.txt

# Juntar todas as linhas em uma
sed ':loop; N; $!b loop; s/\n/ /g' arquivo.txt

# Duplicar cada linha
sed 'p' arquivo.txt
bash — grep
grep -c "erro" log.txt           # conta ocorrências
grep -v "DEBUG" log.txt          # inverte: linhas SEM "DEBUG"
grep -E "erro|falha" log.txt     # regex estendida (ERE)
grep -rn "TODO" ./src/           # recursivo com nº de linha
grep -i "erro" log.txt           # case-insensitive
grep -A2 -B2 "erro" log.txt      # 2 linhas antes e depois
